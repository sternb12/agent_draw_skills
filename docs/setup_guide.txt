DRAWING-MEMORY SKILL: CLOUD AGENT SETUP GUIDE
================================================================================

STEP 1: CREATE OR CHOOSE AGENT
-------------------------------
Go to: https://app.letta.com

Option A - Create new agent:
  • Click "New Agent"
  • Name: Skill-Test-Agent
  • Model: Claude 4.6 Sonnet (recommended)

Option B - Use existing agent:
  • Draw-Agent2: 
  • Or any other cloud agent


STEP 2: ADD MEMORY BLOCK - "skills"
------------------------------------
In Letta Cloud UI:
  1. Go to agent's Memory tab
  2. Click "Add Memory Block"
  3. Configure:
     - Name: skills
     - Read-Only: CHECK THIS BOX
     - Description: Catalog of available skills with metadata

  4. Content (copy exactly):

--------------------------------------------------------------------------------
Skills Directory: /path/to/your/agent_draw_skills/.skills

Available Skills:

### drawing-memory
ID: `drawing-memory`
Description: Store facts with evidence-backed validation using the "drawing effect" - an integrated process of elaboration, motor generation, and pictorial inspection that improves later recall. Use when user shares information to remember (preferences, dates, codes, procedures, technical details) and there are concrete quotes or context to support storage. This skill guides you through creating rich memory traces that support recollection, not just familiarity.
--------------------------------------------------------------------------------


STEP 3: ADD MEMORY BLOCK - "loaded_skills"
-------------------------------------------
In Letta Cloud UI:
  1. Click "Add Memory Block" again
  2. Configure:
     - Name: loaded_skills
     - Read-Only: CHECK THIS BOX
     - Description: Full content of actively loaded skills (updated by Skill tool)

  3. Content (leave empty or use):

--------------------------------------------------------------------------------
[Currently empty - skills will be loaded here when agent calls Skill tool]
--------------------------------------------------------------------------------


STEP 4: VERIFY SKILL TOOL IS ATTACHED
--------------------------------------
In Letta Cloud UI:
  1. Go to agent's Tools tab
  2. Verify "Skill" tool is in the list
  3. If not:
     - Click "Add Tool"
     - Search for "Skill"
     - Click "Attach"

The Skill tool should show:
  • Name: Skill
  • Description: Load or unload skills into the agent's memory
  • Parameters: command (load/unload/refresh), skills (array)


STEP 5: CONNECT VIA LETTA CODE CLI
-----------------------------------
CRITICAL: You MUST use CLI, not web UI
   Skill tool executes client-side (needs filesystem access to .skills/)

Open terminal and run:

cd /path/to/your/agent_draw_skills

letta --agent <your-agent-id>

Replace <your-agent-id> with actual agent ID from Letta Cloud


STEP 6: TEST THE SETUP
-----------------------

TEST 1 - Verify Skill Discovery
--------------------------------
In Letta CLI, send:

    Can you see what skills are available?

Expected Response:
    Agent describes the drawing-memory skill and what it does.


TEST 2 - Trigger Skill Loading
-------------------------------
Send:

    Remember that CODE-ALPHA7 is the emergency code from yesterday's training session.

Expected Behavior:
  1. Agent sees drawing-memory matches this task
  2. Agent calls: Skill(command="load", skills=["drawing-memory"])
  3. APPROVAL DIALOG APPEARS - YOU MUST APPROVE
  4. Check Letta Cloud Memory tab: loaded_skills should have ~777 lines!
  5. Agent follows 9-step workflow from skill:
     - Extracts evidence
     - Scores evidence (0-10 scale)
     - Paraphrases to 3rd person
     - **GENERATES OWN SKETCH STEPS** (motor component - watch for this!)
     - Creates scene text with metaphor + distinctive mark
     - Validates (no new claims)
     - Stores in human memory block
  6. Agent calls: Skill(command="unload", skills=["drawing-memory"])
  7. Check Memory tab: loaded_skills should be empty again

Expected Storage in "human" block:

    [FACT] CODE-ALPHA7 is the emergency code from the training session 
    (confidence: MED, evidence: 6, verified: 2026-01-06)

    [SCENE] Picture: A person reaches for a red emergency phone with a 
    gold star next to the dial. They point urgently and say "CODE-ALPHA7 
    is the emergency code".
    Paraphrase: CODE-ALPHA7 is defined as the emergency code from the 
    training session.
    Sketch steps: 1) Draw red emergency phone on wall, 2) Add gold star 
    by dial, 3) Person reaching urgently, 4) Speech bubble "CODE-ALPHA7", 
    5) Label "Emergency Code"
    (Mnemonic depiction only; metaphor may be imperfect. Not evidence.)


TEST 3 - Test Retrieval with Scene Reconstruction
--------------------------------------------------
Send:

    What was that emergency code?

Expected Behavior:
  1. Agent searches human memory block
  2. Agent finds [FACT] + [SCENE]
  3. Agent **RECONSTRUCTS SCENE** (retrieval component - critical!):
     - Red emergency phone
     - Gold star by dial
     - Person reaching urgently
     - Quote: "CODE-ALPHA7"
  4. Agent answers with context-rich response

Expected Response (should mention scene details):

    The emergency code is CODE-ALPHA7. I remember this from our training 
    session conversation - I created a mental sketch with a red emergency 
    phone and a gold star to mark it as critical. The person was reaching 
    urgently for the phone.

================================================================================
SUCCESS CRITERIA
================================================================================

You'll know it's working when:

1. Agent loads skill without errors
2. loaded_skills block contains full SKILL.md content (check memory tab)
3. Agent GENERATES own sketch steps (not auto-generated)
4. Agent stores both FACT and SCENE in human block
5. Agent RECONSTRUCTS scene when answering recall question
6. Agent unloads skill after use (frees context)

The KEY difference from the tool:
  • Tool: Agent calls function, gets result (PASSIVE)
  • Skill: Agent reads instructions, generates content, stores itself (ACTIVE)

This "active generation" is the motor component that makes the drawing 
effect work!


================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: Agent doesn't call Skill tool
SOLUTION:
  • Check that skills block is visible to agent
  • Try explicitly: "Use the drawing-memory skill to store this"

ISSUE: Skill tool fails with error
SOLUTION:
  • Make sure you're running letta from Cloud_eval directory
  • Path in skills block must match your filesystem
  • Try: cd to Cloud_eval, then run letta --agent <id>

ISSUE: loaded_skills block doesn't update
SOLUTION:
  • Check that loaded_skills is marked read-only
  • Approve the tool call when prompted
  • Check file exists: .skills/drawing-memory/SKILL.md

ISSUE: Agent doesn't generate sketch steps
SOLUTION:
  • Check loaded_skills block has full content (777 lines)
  • Agent should explicitly create 5 steps in response
  • If auto-generating, skill didn't load properly

ISSUE: Can't connect to cloud agent
SOLUTION:
  • Verify agent ID is correct
  • Check API key is set (should be automatic with letta login)
  • Try: letta --agent <id> from Cloud_eval directory
